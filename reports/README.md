# 手机品牌分析报告

本目录包含手机品牌场景及差评分析报告的生成脚本和生成的报告文件。

## 📁 文件说明

### 脚本文件
- `phone_brand_analysis_v2.py` - 手机品牌差评分析报告生成器

### 报告文件
- `手机品牌场景及差评分析日报_2026-04-27.md` - 2026年4月27日的分析报告

## 🚀 使用方法

### 运行脚本生成新报告

```bash
# 进入reports目录
cd /workspace/projects/reports

# 运行生成脚本
python phone_brand_analysis_v2.py
```

脚本会自动：
1. 分析6个手机品牌（苹果、华为、小米、OPPO、荣耀、vivo）
2. 从公开渠道检索用户反馈
3. 分类整理问题（通信类、系统应用类、硬件品质类）
4. 生成完整的分析报告

### 自定义报告

如需修改分析的品牌、问题类别或数据来源，可以编辑 `phone_brand_analysis_v2.py` 文件中的配置：

```python
# 修改品牌列表
self.brands = ["苹果", "华为", "小米", "OPPO", "荣耀", "vivo"]

# 修改问题类别
self.categories = {
    "通信类": ["WIFI", "5G", "4G", "断流", "信号差", "通话问题", "蓝牙", "NFC", "GPS", "定位"],
    "系统应用类": ["系统更新", "续航", "发热", "卡顿", "闪退", "杀后台", "BUG", "功能异常"],
    "硬件品质类": ["屏幕", "绿线", "漏液", "花屏", "电池", "铰链", "结构件", "拍照", "影像"]
}
```

## 📊 报告内容

生成的报告包含以下章节：

1. **统计周期与生成日期** - 明确数据的时间范围
2. **数据来源说明** - 说明数据的获取渠道和方法
3. **整体数据概览** - 各品牌整体反馈情况和问题类别分布
4. **各品牌详细分析** - 每个品牌的详细问题分析
   - 数据来源概况
   - 通信类问题分析
   - 系统/应用类问题分析
   - 硬件品质类问题分析
   - TOP问题排行
   - 典型案例
5. **跨品牌差异分析** - 不同品牌在各问题类别上的对比
6. **趋势总结** - 新增/恶化问题和缓解/下降问题
7. **需重点关注的问题** - 跨品牌共性和品牌特定问题
8. **数据局限性说明** - 数据的局限性和使用建议

## 📝 注意事项

1. **数据来源**: 报告基于公开网络信息检索生成
2. **数据时效性**: 日报仅覆盖最近1天的数据
3. **数据验证**: 建议结合官方数据进行交叉验证
4. **报告仅供参考**: 本报告仅作为舆情监测的参考
